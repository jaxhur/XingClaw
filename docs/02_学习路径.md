# 项目入口

**项目入口**

- src/coding_agent/cli.py：编程 Agent 命令行入口
- src/im/cli.py：飞书 IM 桥接入口



# 项目启动方式

当前项目启动的前提是执行了`pip install -e ".[dev]"`将当前项目作为一个包安装了

原因

- 大量使用包内相对导入`from .cli import main`，这种写法要求它以“包”的身份运行。如果你直接执行某个 .py 文件，Python 会把它当普通脚本，不知道它属于 coding_agent 包，相对导入就容易失败。





**项目启动方式**

- coding_agent：命令行编程 Agent，入口文件是 src/coding_agent/cli.py

  ```
  # 使用默认参数，等价于python -m coding_agent --mode interactive
  python -m coding_agent
  # 更完整、更明确的 CLI 启动方式
  python -m coding_agent --mode interactive --provider openai-standard --model-id deepseek-v4-pro
  ```

- im：飞书 IM 桥接服务入口文件是 src/im/cli.py，接收飞书消息，然后把消息转给 coding_agent。

  ```
  # 启动 IM 桥接服务
  python -m im
  python -m im --transport longconn 
  ```

- Windows 开发脚本

  - 加载 .env.ps1
  - 检查项目有没有 pip install -e ".[dev]"
  - 从环境变量读取 FEISHU_APP_ID、FEISHU_APP_SECRET、XINGCLAW_PROVIDER、XINGCLAW_MODEL_ID
  - 最后拼出并执行`python -m im --platform feishu --transport longconn ...`

  ```
  .\dev.ps1 -Mode im -Transport longconn
  ```

- 安装后命令：pyproject.toml

  - xingclaw等价于python -m coding_agent
  - xingclaw-im等价于python -m im

  ```
  xingclaw = "coding_agent.cli:main"
  xingclaw-im = "im.cli:main"
  ```

原因是**没有告诉它用哪个模型**，coding_agent 的 CLI 默认 provider=None、model_id=None。它会尝试从这些地方找模型，但是当前 .xingclaw 目录里没有 settings.json，所以它找不到模型。命令行参数：--provider、--model-id

1. .xingclaw/settings.json
2. 已存在的 --session-id 对应的历史会话 meta
3. 传入的底层 model 对象

所以需要执行

```
python -m coding_agent --provider openai-standard --model-id deepseek-v4-pro
# 或者
xingclaw --provider openai-standard --model-id deepseek-v4-pro
```

``.\dev.ps1 -Mode im -Transport longconn` 能跑，是因为 dev.ps1加载 .env.ps1，读取XINGCLAW_PROVIDER和XINGCLAW_MODEL_ID，拼成

```
python -m im --provider xxx --model-id xxx ...
```

`python -m coding_agent` 和 `python -m im` 里的`coding_agent、im `指的是**Python 包名**，执行`python -m coding_agent`，Python 会**在 sys.path 里找一个叫 coding_agent 的包**。因为已经 pip install -e ".[dev]" 或 uv sync 过，所以 src/ 下的包能被 Python 找到。找到之后，如果它是一个包，Python 会执行`coding_agent/__main__.py`里面只有，所以最终还是进``coding_agent.cli.main()``。

```
from .cli \
import main raise SystemExit(main())
```

`python -m im`同理，Python 找 im 包，然后执行 src/im/__main__.py，最后进入 src/im/cli.py。





# 调用链路

**调用链路**

CLI模式：

```
coding_agent.cli.main()
  -> create_agent_session()
  -> AgentSession
  -> runner.run()
  -> run_interactive() / run_print() / run_rpc()
  -> AgentSession.prompt()
  -> agent_core.Agent.prompt()
  -> agent_core.agent_loop.run_agent_loop()
  -> ai.stream.stream_simple()
  -> provider API
  -> 工具调用 / 最终回复
```

IM 飞书模式：

```
im.cli.main()
  -> FeishuAdapter
  -> IMService
  -> webhook / longconn 接收消息
  -> SessionRouter 找到频道对应 session
  -> create_agent_session()
  -> AgentSession.prompt()
  -> 同一套 agent_core 主循环
```



# 如何学习这个项目

**学习顺序**

- 先看文档总览：docs/00_XingClaw总览.md，把项目拆成 4 层：ai、agent_core、coding_agent、im。

读源码顺序：

1. examples/quickstart.py：最小 LLM 调用，知道项目怎么跑起来。
2. src/ai/types.py：消息、模型、工具调用这些核心数据结构
3. src/ai/stream.py：统一调用不同模型 Provider
4. src/agent_core/agent.py：Agent 对外暴露的 prompt()、continue_run()
5. src/agent_core/agent_loop.py：Agent 循环，核心的“模型回复 -> 工具调用 -> 再问模型”的循环。
6. src/coding_agent/factory.py：编程 Agent 是怎么组装模型、工具、system prompt、MCP、skills
7. src/coding_agent/agent_session.py
   看会话层：持久化、上下文压缩、重试、事件订阅。
8. src/im/service.py 和 src/im/session_router.py：把飞书消息转成 AgentSession.prompt()。
